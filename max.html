Linear Data Structures

Sequential data organization and access patterns

🔗

Singly Linked List

Time: O(n) search | O(1) insert/delete at head
Space: O(n)

Key Features

  • Each node contains data + next pointer
  • Dynamic size allocation
  • Sequential access only
  • No memory waste (unlike arrays)
  • Cache locality poor compared to arrays

Common Operations

  • Insert at head: O(1)
  • Insert at tail: O(n) or O(1) with tail pointer
  • Delete by value: O(n)
  • Search: O(n)

Applications

  • Implementation of stacks and queues
  • Polynomial representation
  • Undo functionality in applications
⛓️

Doubly Linked List

Time: O(n) search | O(1) insert/delete at known position
Space: O(n) + extra pointer per node

Key Features

  • Each node has prev + data + next
  • Bidirectional traversal
  • Easier deletion (no need for previous node)
  • More memory per node
  • Can traverse backwards efficiently

Operations Advantage

  • Delete node given reference: O(1)
  • Insert before given node: O(1)
  • Reverse traversal: O(n)
  • Find previous node: O(1)
🎯

Dummy Headed Doubly Linked List

Time: Same as doubly linked list
Space: O(n) + 2 dummy nodes

Key Features

  • Dummy head & tail simplify operations
  • No null pointer checks needed
  • Cleaner insertion/deletion code
  • Edge cases handled automatically
  • Circular with dummy nodes

Implementation Benefits

  • Eliminates special cases for empty list
  • Uniform insertion/deletion logic
  • Reduces bugs in boundary conditions
📚

Stack (LIFO)

Time: O(1) for all operations
Space: O(n)

Key Features

  • Last In, First Out (LIFO)
  • Only access to top element
  • Can be implemented with array or linked list
  • Perfect for recursive algorithms
  • Natural for parsing expressions

Core Operations

  • Push: Add element to top
  • Pop: Remove top element
  • Peek/Top: View top element
  • IsEmpty: Check if stack empty

Applications

  • Function call management
  • Expression evaluation
  • Balanced parentheses checking
  • Undo operations
  • DFS traversal
🎫

Queue (FIFO)

Time: O(1) for all operations
Space: O(n)

Key Features

  • First In, First Out (FIFO)
  • Access at both ends (front & rear)
  • Circular queue for efficient array implementation
  • Priority queue variant available
  • Double-ended queue (deque) extension

Core Operations

  • Enqueue: Add to rear
  • Dequeue: Remove from front
  • Front: View front element
  • IsEmpty: Check if queue empty

Applications

  • BFS traversal
  • Process scheduling
  • Print job management
  • Buffer for data streams
  • Level order tree traversal
🏗️

Two Dimensional Array

Time: O(1) access | O(n²) for full traversal
Space: O(n×m)

Key Features

  • Matrix representation (rows × columns)
  • Row-major or column-major storage
  • Constant time access with indices
  • Cache-friendly sequential access
  • Fixed size after declaration

Common Operations

  • Access: arr[i][j] in O(1)
  • Matrix multiplication: O(n³)
  • Transpose: O(n²)
  • Search in sorted matrix: O(n+m)

Applications

  • Image processing
  • Game boards (chess, tic-tac-toe)
  • Mathematical matrices
  • Dynamic programming tables
  • Adjacency matrix for graphs

Tree Data Structures

Hierarchical data organization with parent-child relationships

🌳

Binary Tree

Time: O(n) traversal | O(log n) balanced operations
Space: O(h) where h is height

Key Features

  • Each node has ≤ 2 children
  • Recursive structure perfect for divide-and-conquer
  • Height determines performance characteristics
  • Can be complete, full, or perfect
  • Foundation for many other tree types

Traversal Methods

  • Inorder: Left → Root → Right
  • Preorder: Root → Left → Right
  • Postorder: Left → Right → Root
  • Level-order: BFS traversal

Applications

  • Expression trees
  • Decision trees
  • File system hierarchy
  • Huffman coding
🔍

Binary Search Tree

Time: O(log n) average | O(n) worst case
Space: O(h) recursion stack

Key Features

  • Left < Root < Right property
  • Inorder traversal gives sorted sequence
  • Self-balancing variants exist (AVL, Red-Black)
  • Efficient searching and sorting
  • Can become unbalanced (worst case: linked list)

Core Operations

  • Search: Compare and go left/right
  • Insert: Find position and add
  • Delete: Handle 0, 1, or 2 children cases
  • Find min/max: Go leftmost/rightmost
  • Successor/Predecessor finding

Implementation Notes

  • Deletion with 2 children: use inorder successor
  • Validate BST: use min/max bounds, not just adjacent
  • Balance factor affects performance significantly
🌴

General Tree (N-ary Tree)

Time: O(n) operations depend on structure
Space: O(h×w) where w is width

Key Features

  • Any number of children per node
  • More flexible than binary trees
  • Can represent hierarchical data naturally
  • Implementation via array of children or linked list
  • Used in file systems and organizational charts

Common Operations

  • Add child to node
  • Remove child from node
  • Find path from root to node
  • Calculate tree depth/height
  • Count nodes at each level

Applications

  • File directory structures
  • Organization hierarchies
  • Abstract syntax trees
  • Game decision trees
  • XML/HTML document structure

Graph & Advanced Structures

Complex relationships and specialized data organization

📊

Graph

Time: O(V + E) traversal | O(V²) or O(E) space depends on representation
Space: O(V + E) adjacency list | O(V²) adjacency matrix

Key Features

  • Vertices (nodes) and Edges (connections)
  • Directed vs Undirected
  • Weighted vs Unweighted
  • Can have cycles (unlike trees)
  • Multiple representations possible

Traversal Algorithms

  • DFS: Stack/Recursion - cycle detection
  • BFS: Queue - shortest path (unweighted)
  • Dijkstra: Shortest path (weighted)
  • Topological Sort: Order for DAGs

Representation Trade-offs

  • Adjacency Matrix: O(1) edge check, O(V²) space
  • Adjacency List: O(V+E) space, better for sparse graphs
  • Edge List: Simple but less efficient for queries

Applications

  • Social networks
  • GPS navigation
  • Web page linking
  • Dependency resolution
  • Network routing
⛰️

Heap (Priority Queue)

Time: O(log n) insert/delete | O(1) peek
Space: O(n)

Key Features

  • Complete binary tree property
  • Max-Heap: Parent ≥ children
  • Min-Heap: Parent ≤ children
  • Array implementation with index calculations
  • Efficient priority-based operations

Core Operations

  • Insert: Add at end, bubble up
  • Extract: Remove root, bubble down
  • Heapify: Build heap from array O(n)
  • Parent: index/2, Children: 2i, 2i+1

Applications

  • Priority queues
  • Heap sort algorithm
  • Dijkstra's shortest path
  • A* pathfinding algorithm
  • K largest/smallest elements
  • Median finding

Hash-Based Structures

Fast access through key-value mapping

#️⃣

Hash Table

Time: O(1) average | O(n) worst case
Space: O(n)

Key Features

  • Hash function maps keys to indices
  • Fast average-case performance
  • Collision resolution strategies needed
  • Load factor affects performance
  • Dynamic resizing maintains efficiency

Collision Resolution

  • Chaining: Linked lists at each bucket
  • Open Addressing: Linear/quadratic probing
  • Double Hashing: Second hash function
  • Load Factor: n/m (entries/buckets)

Design Considerations

  • Good hash function: uniform distribution
  • Resize when load factor > 0.75
  • Handle hash function collisions gracefully
  • Consider cryptographic vs non-cryptographic hashing

Applications

  • Database indexing
  • Caching systems
  • Symbol tables in compilers
  • Unique data detection
  • Password storage (with salting)

Advanced Concepts

Fundamental programming techniques and paradigms

🔄

Recursion

Time: Varies by problem
Space: O(depth) call stack

Key Features

  • Function calls itself
  • Must have base case to prevent infinite recursion
  • Recursive case moves toward base case
  • Uses call stack for state management
  • Can often be converted to iterative solution

Design Pattern

  • Base Case: Simple, directly solvable
  • Recursive Case: Break down problem
  • Combine: Build solution from sub-solutions
  • Progress: Each call gets closer to base

Common Patterns

  • Tree traversal: Process node, recurse on children
  • Divide & Conquer: Split, solve, combine
  • Backtracking: Try, recurse, undo if needed
  • Dynamic Programming: Memoize recursive calls

Applications

  • Tree and graph traversals
  • Factorial, Fibonacci calculations
  • Parsing nested structures
  • Maze solving
  • Permutations and combinations
  • Quick sort, merge sort
