Binary Search Tree

Time: O(log n) avg | O(n) worst
Space: O(h) recursion stack
  • Left < Root < Right property
  • In-order traversal gives sorted sequence
  • Self-balancing variants: AVL, Red-Black
  • Deletion cases: 0, 1, or 2 children
  • Find successor/predecessor efficiently

Key Points

  • Validate: use min/max bounds
  • Balance is crucial for performance
  • Skewed tree = linked list performance

Applications

  • Database indexing
  • Priority queues
  • Sorted collections

Graph

Time: O(V + E) traversal
Space: O(V) for adjacency
  • DFS: Stack/Recursion - cycle detection
  • BFS: Queue - shortest path
  • Directed vs Undirected
  • Weighted vs Unweighted
  • Representation: Adjacency List/Matrix
  • Topological sort for DAGs

Key Points

  • Choose representation wisely
  • BFS for shortest path (unweighted)
  • DFS for connected components

Applications

  • Social networks
  • Route planning
  • Dependency resolution

Hash Table

Time: O(1) avg | O(n) worst
Space: O(n) storage
  • Hash Function: Maps keys to indices
  • Collision Resolution: Chaining vs Open Addressing

  • Trade memory for speed
  • Hash function quality is crucial
  • Dynamic resizing maintains performance

Applications

  • Database indexing
  • Caching systems
  • Symbol tables

Red-Black Tree

Time: O(log n) guaranteed
Space: O(n) + color bit
  • Self-balancing BST variant
  • Nodes colored Red or Black
  • Root and leaves (NIL) are black
  • Red nodes have black children
  • Same black height on all paths
  • Rotations maintain balance

Key Points

  • Guarantees O(log n) operations
  • Used in many standard libraries
  • Less rigid than AVL trees

Applications

  • std::map in C++
  • TreeMap in Java
  • Process scheduling

B-Tree

Time: O(log n) with high branching
Space: O(n) with better locality
  • Multi-way search tree
  • Each node has many keys/children
  • All leaves at same level
  • Minimizes disk I/O operations
  • Order m: max m-1 keys per node
  • Self-balancing with splits/merges

Key Points

  • Optimized for external memory
  • Reduces tree height significantly
  • Complex insertion/deletion

Applications

  • Database indexing
  • File systems
  • Large dataset storage
